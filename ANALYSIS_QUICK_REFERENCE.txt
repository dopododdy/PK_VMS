Two separate files control blood test ordering:

1. lab.html (Comparative Trends display)
   → Uses hardcoded PARAM_ORDER array
   → Fixed order, can't be changed by users
   → ALP always 3rd in Chemistry section

2. set-lab.html (Parameter management)
   → Uses database order_index field
   → Users can drag-and-drop to reorder
   → Changes NOT reflected in lab.html

Result: Possible confusion - two different orders!

Q: Why doesn't my parameter appear in Comparative Trends?
A: It might not be in PARAM_ORDER array. Check lab.html line 279-287.
   If not there, it will be sorted to the bottom.
   
Q: Why did I reorder parameters in set-lab.html but Comparative Trends didn't change?
A: lab.html doesn't use order_index. It uses hardcoded PARAM_ORDER.
   Changes in set-lab.html only affect set-lab.html's display.
   
Q: Can I have different ordering in lab.html and set-lab.html?
A: Yes, you will have different ordering because they use different systems.
   This is a design issue to be aware of.
   
Q: Where do I change the ALP order?
A: To change in Comparative Trends:
   → Edit PARAM_ORDER array in lab.html (line 279-287)
   → Recompile/redeploy the file
   
   To change in set-lab.html sidebar:
   → Drag parameter in UI
   → Changes save automatically to database
   
Q: Is there a way to sync both orderings?
A: Not currently. You would need to modify lab.html to use order_index
   from the database instead of the hardcoded PARAM_ORDER array.
